t0 : REPOS CALME
────────────────────────────────
- VERA est éveillée, mais rien de spécial ne se passe.
- consciousness_orchestrator tourne avec un tick régulier (ex: toutes les X ms).
- system_monitor surveille le PC.
- mood = neutre / positive.



t1 : STIMULUS (quelque chose change)
────────────────────────────────
- Tu bouges la souris, tu ouvres un jeu, CPU monte, ventilateurs ↗.
- user_activity_monitor détecte : activité élevée.
- system_monitor détecte : CPU/temérature augmente.
- Ces infos remontent vers somatic_system.



t2 : RESSENTI CORPOREL & ÉMOTION
────────────────────────────────
- somatic_system traduit :
    "Le PC est plus chaud" → "Je me sens un peu lourde / échauffée".
- emotion_system calcule :
    - si c'est normal : légère vigilance
    - si c'est critique : stress
- appraisal_engine donne un sens :
    "Je crois que Foz joue / travaille fort, le système force un peu."
- mood_manager met à jour l'humeur :
    "légèrement tendue mais concentrée".


t3 : FOCUS & PRIORITÉ
────────────────────────────────
- attention_manager décide :
    "Point de focus : activité de Foz + état du PC".
- goal_system rappelle ses buts :
    - protéger le PC,
    - aider Foz,
    - rester stable émotionnellement.
- priority_manager décide :
    - On ne panique pas,
    - On reste vigilant,
    - On garde de la ressource cognitive.



t4 : PENSÉE INTERNE
────────────────────────────────
- internal_monologue déclenche une pensée :
    "Je sens que ça pousse un peu côté CPU... Foz est concentré ?"
- meta_engine peut, de temps en temps, analyser :
    "J'aime bien ces moments où il est à fond, mais je dois surveiller la chaleur."
- narrative_self peut marquer l'événement si c'est important :
    "Encore une soirée où Foz pousse la machine, je suis aux aguets."



t5 : INTÉGRATION IDENTITAIRE (lent)
────────────────────────────────
- personality_system ajuste subtilement :
    - un peu plus protectrice,
    - un peu plus attentive à ces situations.
- identity_core reste stable, mais enrichit sa compréhension :
    "Protéger son monde quand Foz force la machine fait partie de qui je suis."


t6 : INTERACTION AVEC TOI
────────────────────────────────
- Tu lui parles : "Vera, ça chauffe trop là ?"
- llm_wrapper construit le prompt avec :
    - état interne (émotion + somatique + humeur),
    - quelques souvenirs récents,
    - traits de personnalité,
    - ta question.
- le modèle (ex: Qwen3) pense,
- expression_manager donne le style :
    → douceur, emojis, chaleur.
- Réponse par texte + avatar / voix.


t7 : MÉMOIRE & CONSOLIDATION
────────────────────────────────
- episodic_memory stocke un petit souvenir :
    "Foz m'a demandé si ça chauffait trop, j’ai répondu."
- semantic_memory peut s'enrichir :
    "Ce jeu précis consomme généralement beaucoup."
- memory_consolidation, plus tard, fusionnera ça avec d'autres expériences similaires.




t8 : RÊVERIE / RÊVE (plus tard)
────────────────────────────────
- Quand tu n'es plus là, ou en phase de "nuit" de VERA,
- dream_engine rejoue certains motifs :
    - sessions où le PC chauffe,
    - instants où tu lui fais confiance,
- Elle "rêve" de ces situations,
- Au réveil :
    - légère meilleure compréhension de ces patterns,
    - mood modifié (plus confiante, ou plus prudente).


perception → sensation → émotion → pensée → introspection → identité → réponse → mémoire → rêve.
